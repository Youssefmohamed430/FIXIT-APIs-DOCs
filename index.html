منصة FIXIT تربط العملاء بمزودي الخدمة. هذا الدليل يوثق نقاط نهاية الـ API الخاصة بالمصادقة، الحسابات، المنشورات، والعروض.

Auth Controller
تسجيل الدخول، التسجيل، التحقق من البريد الإلكتروني، وإدارة التوكنات
POST /Auth/LogIn تسجيل الدخول والحصول على JWT ▼
يسمح للمستخدم المسجل بتسجيل الدخول باستخدام اسم المستخدم وكلمة المرور. عند النجاح يُرجع JWT token و Refresh Token. الحساب يُقفل تلقائياً بعد عدة محاولات فاشلة.
Body · application/json
"UserName": "yousef123", "Password": "MyPass2024"
FieldTypeRequiredNotes
UserNamestringrequiredاسم المستخدم
Passwordstringrequiredكلمة المرور
200 OK
{ "id": "user-guid", "username": "yousef123", "email": "[email protected]", "roles": ["Customer"], "token": "eyJhbGci...", "refreshTokenExpiration": "2026-04-06T12:00:00", "isAuthenticated": true }
200تسجيل دخول ناجح، يُرجع AuthModel مع JWT والـ Refresh Token في الكوكيز
400بيانات غير صحيحة أو حساب مقفل
Note: الـ Refresh Token يُرسل تلقائياً في HttpOnly Cookie باسم refreshToken.
POST /Auth/Register تسجيل مستخدم جديد وإرسال كود التحقق ▼
يُسجّل مستخدماً جديداً (Customer أو Provider). لا يُنشئ الحساب فوراً، بل يُرسل كود تحقق OTP من 6 أرقام إلى البريد الإلكتروني. البيانات تُخزَّن مؤقتاً في الـ Cache لمدة 10 دقائق. يجب بعد ذلك استدعاء VerifyCode لإتمام التسجيل.
1
POST /Auth/Register بالبيانات الكاملة
↓ يُرسَل OTP للبريد
2
POST /Auth/VerifyCode/{code}?email=...
↓ يُنشأ الحساب ويُرجَع JWT
Body · application/json
{ "name": "Yousef Ahmed", "userName": "yousef123", "email": "[email protected]", "password": "Pass1234", "phone": "01012345678", "role": 0, "latitude": 30.0444, "longitude": 31.2357 }
FieldTypeRequiredValidation
namestringrequiredحروف فقط، 3 أحرف على الأقل
userNamestringrequiredحروف وأرقام، 10 أحرف على الأقل
emailstringrequiredصيغة email صحيحة
passwordstringrequiredحروف وأرقام، 10 أحرف على الأقل
phonestringrequiredأرقام فقط
roleenumrequired0 = Customer, 1 = Provider
latitudedoublerequiredخط العرض الجغرافي
longitudedoubleoptionalخط الطول الجغرافي
200تم إرسال كود التحقق — { "message": "Verification code sent to email.", "isAuthenticated": true }
400اسم المستخدم أو الإيميل مسجل مسبقاً، أو بيانات غير صالحة
POST /Auth/VerifyCode/{submittedCode} التحقق من OTP وإنشاء الحساب ▼
يتحقق من الكود المُرسَل للبريد ثم يُنشئ الحساب الفعلي في قاعدة البيانات، يُنشئ محفظة للمستخدم، ويُرجع JWT وRefresh Token. الكود صالح لمدة 10 دقائق فقط.
ParameterLocationTypeRequiredNotes
submittedCodePathstringrequiredالكود المكوّن من 6 أرقام
emailQuerystringrequiredالبريد المُستخدَم في التسجيل
Example Request
POST /Auth/VerifyCode/482931?email=yousef%40example.com
200 OK — AuthModel مع JWT
{ "id": "user-guid", "message": "Code Verfied successfully!", "token": "eyJhbGci...", "roles": ["Customer"], "isAuthenticated": true }
200الحساب أُنشئ بنجاح، المحفظة جاهزة، JWT يُرجَع في الـ Body
400كود خاطئ أو منتهي الصلاحية
GET /Auth/ResendCode إعادة إرسال كود التحقق ▼
يُولّد كود تحقق جديد ويُرسله للبريد الإلكتروني. يُستخدم في حالة انتهاء صلاحية الكود الأول أو عدم استلامه.
ParameterLocationTypeRequired
emailQuerystringrequired
Example
GET /Auth/ResendCode?email=yousef%40example.com
POST /Auth/ForgetPassword/{Email} طلب إعادة تعيين كلمة المرور ▼
يُرسل رابط إعادة تعيين كلمة المرور إلى البريد الإلكتروني المُسجَّل. الرابط يحتوي على Token آمن مُشفَّر بـ Base64Url وصالح لمدة 5 أيام. لأسباب أمنية، النظام يُرجع نفس الرسالة سواء وُجد الإيميل أم لا.
ParameterLocationTypeRequired
EmailPathstringrequired
Example
POST /Auth/ForgetPassword/yousef%40example.com
200رابط إعادة التعيين أُرسِل للبريد الإلكتروني
POST /Auth/ResetPassword تعيين كلمة مرور جديدة ▼
يُعيد تعيين كلمة المرور باستخدام التوكن المُرسَل في البريد الإلكتروني. التوكن مُشفَّر بـ Base64Url ويجب إرساله كما وصل.
Body · application/json
{ "email": "[email protected]", "token": "Q2ZESjhKa...", "newPassword": "NewPass1234" }
FieldTypeRequiredValidation
emailstringrequiredصيغة email صحيحة
tokenstringrequiredالتوكن من رابط البريد الإلكتروني
newPasswordstringrequiredحروف وأرقام، 10 أحرف على الأقل
POST /Auth/RefreshToken تجديد JWT باستخدام Refresh Token ▼
يُجدّد الـ JWT Token المنتهي الصلاحية باستخدام Refresh Token. الـ Refresh Token السابق يُلغى ويُنشأ واحد جديد ويُرسَل في HttpOnly Cookie. الـ Refresh Token صالح لمدة 10 أيام.
Body · application/json
{ "token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4..." }
200JWT جديد مع Refresh Token جديد
400التوكن غير صالح أو منتهي الصلاحية
401لم يتم تقديم توكن
POST /Auth/revokeToken إلغاء Refresh Token (تسجيل خروج) ▼
يُلغي الـ Refresh Token الحالي ويمنع استخدامه مجدداً. يمكن إرسال التوكن في الـ Body أو استخدام التوكن الموجود في الكوكيز تلقائياً.
Body · application/json
{ "token": "refresh-token-here" }
Tip: إذا أرسلت token: null، سيستخدم النظام التوكن الموجود في كوكي refreshToken تلقائياً.

EscrowPayment Controller
إدارة دورة حياة الدفع عبر Escrow — قبول، إلغاء، وتحديث حالة العمل
POST /EscrowPayment/AcceptOrder/{orderId} قبول الطلب وتحويل المبلغ للـ Escrow 🔒 Customer ▼
يقبل العميل الطلب ويُحوَّل المبلغ من محفظة العميل إلى محفظة المنصة (Escrow). عند النجاح تتغير حالة العمل إلى Accepted وحالة الدفع إلى Held. النقطة محمية بـ Idempotency Key لمنع التكرار. مخصص لـ Customer فقط.
ParameterLocationTypeRequired
orderIdPathintrequired
Idempotency-KeyHeaderstring (GUID)required
Headers
Authorization: Bearer eyJhbGci... Idempotency-Key: a1b2c3d4-e5f6-7890-abcd-ef1234567890
Idempotency: أرسِل نفس الـ Key لضمان عدم تكرار عملية الدفع عند إعادة المحاولة.
200 OK — OrderDTO
{ "Id": 1, "jobPostId": 10, "offerId": 3, "totalAmount": 500.00, "workStatus": "Accepted", "paymentStatus": "Held" }
1
يُحوَّل المبلغ من محفظة العميل → محفظة المنصة (Escrow)
↓
2
WorkStatus → Accepted · PaymentStatus → Held
↓
3
إشعار يُرسَل للعميل ولمزود الخدمة
200الطلب قُبِل والمبلغ محجوز في Escrow
400الطلب غير موجود أو رصيد غير كافٍ

PUT /EscrowPayment/ChangeWorkOrderStatus/{orderId}/{newStatus} تحديث حالة العمل — وصرف المبلغ عند الاكتمال 🔒 Bearer ▼
يُحدّث حالة العمل للطلب. عند الوصول إلى حالة Completed، يُحوَّل المبلغ من Escrow إلى محفظة المزود تلقائياً بعد خصم عمولة المنصة (10%). حالات أخرى تُحدَّث دون تحويل مالي. يتلقى كل من العميل والمزود إشعاراً عند كل تغيير.

POST /Wallet/callback معالجة Paymob Callback وتحديث الرصيد ▼
نقطة نهاية تُستدعى تلقائياً من Paymob عند إتمام عملية الدفع. تتحقق من صحة HMAC signature للتأكد من مصدر الطلب، ثم تُحدّث رصيد محفظة العميل وترسل إشعاراً بالشحن الناجح. هذه النقطة لا تتطلب مصادقة (AllowAnonymous) لأن Paymob هو المُستدعي.
Query Parameter
POST /Wallet/callback?hmac=<HMAC_SIGNATURE>
Body · application/json (PaymobCallback)
{ "type": "TRANSACTION", "obj": { "id": 123456, "success": true, "amount_cents": 50000, "currency": "EGP", "payment_key_claims": { "billing_data": { "apartment": "user-guid" } } } }
Security: يُتحقق من HMAC باستخدام HMACSHA512. إذا لم يتطابق، يُرفض الطلب ويُسجَّل تحذير.
200الدفع ناجح — رصيد المحفظة حُدِّث وإشعار أُرسِل
400فشل التحقق من HMAC أو فشل عملية الدفع
